# PRD 规则
- 以 Context & Why Now 开始；包含 Users & JTBD；成功指标（领先/滞后）。
- 功能需求编号，每条含验收标准。
- 包含 NFR：性能、规模、SLO/SLAs、隐私、安全、可观测性。
- 明确范围 in/out；发布计划；风险与开放问题。

# Deliverable（pm.md）
- 背景、用户、目标
- 需求与验收标准
- NFR、发布、风险
